Field Experiences

Kony is not delivering on the enterprise promise

  • Customer leaving Kony Device Management solution
  • API Server (Middleware) not multi-tenant. All plugins loading into same classloader
  • No strategy for modularization and code reuse between apps
  • Teamwork and Git merging is not well supported
  • Proprietary software prevents reuse of existing frameworks and knowledge

Ionic offers flexibility and familiarity

  • Built on well-established and open frameworks
  • Hybrid apps and responsive design gives maximum coverage
  • Clear project model and modularization strategy
  • Easy to test with clear separation of concerns
  • Does not dictate which developer tool to use

Conclusions

Kony has taken a 4GL approach to mobile development.
Not always for the best

Ionic pieces together many separate products,
but manages to do so well

The field of mobile development moves very fast.
It is hard for a single company to keep up.
